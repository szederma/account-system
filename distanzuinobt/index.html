DistanzuinoBT

Meet the DistanzuinoBT, an innovative musical instrument that evolved from the original Distanzuino, designed by Dóra Szederkényi and Mátyás Szederkényi[cite: 1, 3]. Its name combines "distance" and "Arduino," reflecting its core technology[cite: 4]. This instrument offers a truly revolutionary way to create music: simply by moving your hand over a note area, a laser sensor determines the distance, sending this data via Bluetooth to a custom website which then converts it into a musical tone[cite: 8, 9, 15]. The DistanzuinoBT is based on three main parts: its functions, its 3D-printed body and appearance, and the notes or sound it produces[cite: 5].

How DistanzuinoBT Works

A step-by-step look at the technology from gesture to sound.

1

Gesture Detection & Distance Measurement

The user places their hand over a desired note area on one of the instrument's two sides[cite: 9, 7]. A VL53L1X laser sensor measures the distance to the hand[cite: 9, 77]. This sensor uses the Time-of-Flight (ToF) principle, calculating the time it takes for emitted light to hit an object, reflect, and return[cite: 24, 25, 78]. The instrument can measure two distances at the same time due to its dual-sided design[cite: 31].

2

Data Processing & Bluetooth Transmission

The Arduino Nano microcontroller controls the laser sensor and processes the measured distance values[cite: 23]. These distance values are then transmitted by an HM-10 Bluetooth module[cite: 27, 80]. This module uses Bluetooth Low Energy (BLE), which is necessary for the website API[cite: 28]. The Arduino sends the values using `Serial.print` so the Bluetooth module can relay them to the connected website[cite: 29, 30].

3

Web Application Interpretation

The custom-built DistanzuinoBT web application connects to the instrument via the browser's Bluetooth API[cite: 20, 34, 35]. Once connected, it receives the measured distance data from the Bluetooth module[cite: 36]. The website stores and continuously updates these distance values in variables[cite: 37]. Specific tones are triggered based on predefined distance ranges in the code (e.g., if distance1 is between 3cm and 8.99cm, the tone is H1)[cite: 39, 40].

4

Realistic Sound Generation

Using the interpreted distances, the website generates tones[cite: 38]. It defines the corresponding frequencies for each tone[cite: 41]. Users can select which instrument they want to hear (e.g., piano, flute, cello)[cite: 42, 13]. The web app plays the selected instrument’s fundamental frequencies and their associated harmonics simultaneously to create a rich, realistic sound (e.g., Piano C1 might involve frequencies like 133Hz, 258Hz, and 391Hz at specific intensities)[cite: 43, 14]. The sound generation considers the fundamental frequency, overtones (harmonics) and their intensities, and the crucial "attack phase" – how a note's sound evolves over time from its start – to accurately mimic an instrument's unique timbre[cite: 52, 53, 54, 55, 56, 57, 68, 69, 70, 71].

5

Audio Output & User Customization

The final sound is played through the user's device speakers (computer, phone, tablet) or can be routed to headphones or external Bluetooth speakers for an enhanced experience[cite: 19, 47, 84]. On the website, users can adjust the volume for each side of the instrument independently using sliders[cite: 44]. The interface also provides a visual representation, highlighting which notes are currently being played[cite: 45, 46]. The system is designed to be modifiable with a basic understanding of programming[cite: 11].
